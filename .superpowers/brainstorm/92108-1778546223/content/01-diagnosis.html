Where it still feels raw
  1. Control-band crush before selection. Orders shows 6 sibling verbs always (Ready·Post·Reprice·Fulfill·Pick·Cancel). PO: Approve·Receive·Cancel always. Vendor Payables: Approve·Schedule·Pay always. Recovery: 3 always-on control bands.
  2. Selection acknowledged but not promoted. Sticky footer ≠ primary work surface.
  3. PO is two grids stacked. No persistent PO header carried into line editing. No status-aware primary.
  4. Inventory Finder is locked inside Sales. Most operator-loved tool, unreachable from procurement, intake, recovery.
  5. Customer Workspace not durable. Forgets on tab change. No vendor analog.
  6. Focus mode is binary & lonely. Hides ALL other panels including Quick Start — operator loses orientation.
  7. Cmd-K has a JSON payload editor visible by default. Power tool leaks into daily flow.
  8. 13 left-nav lanes presented flat. Real surfaces, but no grouping for muscle memory.
  9. Density inconsistency. Dashboard calm; Sales stacks 3 control bands + 2 grids + side panel.
7 principles for this pass
  1. One status-aware primary action per surface. Secondaries collapse into selection-aware tray / row-action menu.
  2. Pre vs. post-selection clarity. Pre = start work (Quick Start, new-row, filter). Post = the band swaps to selection-aware primary + tray + impact strip.
  3. Persistent context bands. Customer · Vendor · PO identity stays visible while you work inside them.
  4. Inventory Finder is global. Cmd-Shift-F overlay anywhere. Embeddable panel in Sales / Procurement / Intake. Standalone Inventory gets a Finder mode tab.
  5. Selection summary → context strip. Sums + issues + primary + tray + history mini-strip + related-entity peek, beside the grid.
  6. Focus mode keeps the keel. One-line command strip (Cmd-K + 4 launch icons) survives focus. Only grid below maximizes.
  7. Nav grouping without IA churn. Same 13 routes, 5 muted section dividers: Decide · Procure · Sell · Fulfill · Resolve.
