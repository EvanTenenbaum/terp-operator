# Benchmark digest: Linear / Airtable / Shopify (full report in agent transcript; key quotable rules)
1. Density via visual weighting, not data removal (Linear design refresh): central-task elements in focus, orientation elements recede.
2. One primary action per index page, top-right (Polaris resource-index).
3. Saved views are first-class: tabs on the table (Shopify IndexFilters), nameable/shareable (Linear custom views), curated per audience (Airtable).
4. Side peek/sidesheet DEFAULT, full page is the escalation (Airtable record detail's default = sidesheet; Linear peek/Space).
5. Polaris resource-details: two columns, primary content = 2/3, defining info left, status/metadata/summaries right; "Always use the default width."
6. Polaris: avoid modals for resource creation — configure on the created page itself.
7. One primary button per page; secondary actions roll into "More actions" (Polaris); actions reachable at 3 depths: shortcut → context menu → full page (Linear).
8. "Don't place CTAs in the card header"; "Don't allow cards to become so tall that they are difficult to overview" (Polaris card layout).
9. Collapsible groups/tabs for field-heavy records instead of trimming fields (Airtable).
10. "Structure should be felt not seen": dividers only in tables; hierarchy via size/weight/contrast (Linear + Polaris).
11. Home = action queue + small metric summary, NOT analytics: Shopify Home = 4 metrics + open task counts by type; Linear home = Inbox for keyboard triage (snooze/mark-read). Deep data lives in Analytics, elsewhere.
12. Linear Method: "Simple first, then powerful." Airtable: pick layout by job (record review = triage master-detail; record summary; dashboard = exec, "without excessive detail").
13. Polaris: "The admin functions as software rather than a website."
