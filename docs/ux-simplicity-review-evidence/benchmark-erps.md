# ERP benchmark digest (Odoo/NetSuite/ERPNext/BC) — full text in session; key facts
## Related-records placement (the placement-standard calibration)
- Count+navigate: Odoo smart buttons (counts in form HEADER); ERPNext Connections (buried in last tab = worse)
- Persistent side panel: BC FactBoxes (read-mostly context, collapsible, glanceable while working)
- Inline tables behind tabs: NetSuite Related Records subtab = max info, max page weight — the pattern most correlated with "cluttered/slow" complaints
- Distilled rule: inline grid ONLY for data edited as part of the doc (line items); side panel for read-mostly context; count-button/drawer for whole related docs; separate page only when related record has its own lifecycle.
## Lists
- Default density: BC/NetSuite dense (8–12 cols), Odoo/ERPNext sparse (4–8) with opt-in columns; convergent: per-user persisted column choice.
- Odoo omnibox filter-chips + shareable default-able Favorites = fastest filter UX. BC filter pane + named Views (personal-only = noted gap). BC Analysis Mode = explore without leaving the list.
- Inline editing is the EXCEPTION in documents lists; all four prefer select-rows → explicit bulk action.
## Posting
- Shared skeleton: ONE status-gated primary verb, fixed position; param dialog only for real choices (BC Ship/Invoice/Both); Odoo clickable statusbar breadcrumb = best status viz; BC Preview Posting = best confidence-before-commit (show ledger consequences pre-commit).
- NetSuite dual-lane: per-record buttons for exceptions + batch screens (Bill Sales Orders) for throughput.
## Home
- Consensus: home = actionable counts w/ one-click drill-down to filtered worklist (BC cues w/ threshold colors; NetSuite Reminders). KPIs/charts secondary. Role-tailored (BC role centers per profile).
- Anti-pattern documented: unbounded portlet canvas (NetSuite) → slow cluttered homepage.
- Odoo has NO KPI home (app grid); Accounting dashboard = journal-card worklist.
## Criticized weaknesses (calibration)
- Odoo: hidden settings, minimalism hides depth. NetSuite: dense/slow/dated pre-Redwood; vendor conceded clutter via Redwood. ERPNext: polish gaps, confusing default filters. BC: discoverability under breadth (mitigated by Tell Me Alt+Q), weak doc-status viz.
## BC FastTabs = most formalized progressive disclosure: 3 tiers (Promoted on collapsed header / Standard / Additional behind "Show more") on one page.
