## ItemsView — 2 violations

### Finding 1 — Create item band (M)

**File:** `src/client/views/ItemsView.tsx:171`
**Violation:** Bespoke `<div role="region" aria-label="Create new item">` stacked above the
grid with `bg-zinc-50` background, manual cancel/submit buttons, and toast-based validation
(`pushToast('Item name is required.', 'error')`).
**Correct template home:** `templates/FormDialog` — this is a one-shot record entry.
**Size: M** — edit-state hydration from grid row selection, no colocated `*.test.tsx` or
`*.a11y.test.tsx`, toast validation needs converting to inline `error` prop. Recommend a
dedicated PR (`ux/followup-itemsview-create-dialog`).

### Finding 2 — Edit item band (M)

**File:** `src/client/views/ItemsView.tsx:233`
**Violation:** Bespoke `<div role="region" aria-label="Edit item">` with `bg-amber-50`
background, separate edit-state `useState` variables (`editName`, `editCategory`, etc.),
and `openEdit(row)` callback that hydrates state from the selected grid row.
**Correct template home:** `templates/FormDialog` — same pattern as Create.
**Size: M** — can be bundled with Finding 1 in the same PR. Deactivate-item destructive
action also present and would need its own `FormDialog` instance or an `isDestructive` prop.

## CreditReviewView — 1 violation

### Finding 3 — CreditDivergencePanel disclosure band (M)

**File:** `src/client/views/CreditReviewView.tsx:118`
**Violation:** Owner-gated `CreditDivergencePanel` rendered in a bespoke `<div className="border-b border-zinc-200">` wrapper controlled by a hand-rolled `divergenceOpen` state
toggle with manual `aria-expanded` on the trigger button (`aria-label="Divergence report"`).
**Correct template home:** `WorkspacePanel` — this is a repeated in-page work tool, not
a one-shot entry. Wrapping in `WorkspacePanel` would give it the standard collapsible
heading, accessibility contract, and `panelId` persistence for free.
**Size: M** — `CreditDivergencePanel` is a standalone component with its own KPI tile
chrome; wrapping it is ~5-line change in `CreditReviewView.tsx` but a `creditOps.test.tsx`
behavior test must stay green. Recommend PR `ux/followup-creditreview-divergence-panel`.
