What this batch establishes (so subsequent batches can build on it)
  • 14-route nav structure in 5 groups. Reports lane added. Group dividers are 9px uppercase headers — no horizontal rules.
  • Keel pattern: ⌘K + 5 launch chips + health + user. Active chip when on its primary view.
  • Identity ribbon pattern: visible only when entity active. Slim. Holds identity + status + key id + leave/back.
  • Band swap pattern: same physical row toggles between pre-selection (start work) and selection (act on rows). Status-aware primary in green; tray in "More ▾"; ⏱◇⚠⬇ icons on the right.
  • Drawer pattern: peek (280px) · expanded (420px) · full (≥60%) · closed (24px nub). State persisted per route × entity.
  • Below-floor warning: amber inline pill on line + drawer Balance summary. Per-customer toggleable. Never blocking.
  • Inventory Finder occupies right rail when drawer is closed (Sales view only). ⌘⇧F opens the global overlay anywhere.
  • Dashboard introduces "Today focus" strip + KPI ? opens drawer for definitions, KPI value click opens drawer for drilldown.

If batch 1 reads right, batch 2 is next: Intake / Receiving, Purchase Orders (status-aware header + line-add as first row + drawer tabs), and Inventory Finder (standalone Inventory page Finder mode + ⌘⇧F overlay). Then batch 3: Orders queue, Payments + Quick Ledger, Vendor Payables. Then sketches: Fulfillment, Connectors, Recovery, Closeout, Reports.
