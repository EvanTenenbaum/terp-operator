Finder · standalone Inventory spec: The Inventory route now has a Grid / Finder mode toggle. Grid is the dense full-batch table (edit qty / price / lot / markers inline). Finder is the scanning surface — saved slices + natural-language search + facet refinement + compare + match reasons. The just-added row (J2) shows the new add-signal: green tint with a brief +1 ✓ chip that fades to a quieter "just added · in draft" after 2s. The Finder works with or without an active sales order — Add adds to active draft when one exists; otherwise "Copy customer-safe offer" is the operator's path to a quick out-of-band quote. When the operator clicks a batch row (not the +), the drawer opens with that batch's tabs (Movement / Sales / Reservations / Photos / Sourced from PO / Tags / Transfer / History). ⌘⇧F opens the same Finder as an overlay floating over any view.

What batch 2 establishes
  • Status-aware primary patterns: Intake (Mark Ready → Post Receipt), PO (Approve → Receive to Intake → View linked receipts). Primary disabled with plain reason when status is mixed.
  • Persistent PO header in the identity ribbon while editing lines or scanning linked downstream rows.
  • Line-add as the last row of the line grid — no separate strip. Placeholder text gives the pattern; ↵ commits.
  • Drawer Lines tab is the one read/write exception: PO lines edited here because the PO line grid is the PO's primary work surface, not reference content.
  • Impact preview blocks no actions — it makes "what happens if I click this" visible before commit.
  • Linked-intake / Linked-receipts tabs are the procurement trace surface — one glance from PO → intake → receipt → bill → payout.
  • Inventory Finder is reachable three ways: embedded right rail (Sales), standalone Inventory route's Finder-mode toggle, ⌘⇧F floating overlay anywhere. Same finder behavior; the surface differs.
  • Add-signal pattern: brief green tint on the row + a small +1 ✓ chip that fades to "just added · in draft" after ~2s. Non-blocking, non-modal.

If batch 2 lands, batch 3 next: Orders queue (status-aware primaries across draft/confirmed/posted/fulfilled), Payments + Quick Ledger (band-swap on the ledger grid + allocations drawer), Vendor Payables (per-vendor identity + scheduled-event clarity + payout reconciliation drawer). Then sketches: Fulfillment, Connectors, Recovery, Closeout, Reports.
