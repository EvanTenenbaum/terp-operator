Section 3.5 — Functional Coverage Map

The whole manifest, mapped to the canvas. One placement law, no exceptions.

The placement law (one rule, eight slots)
Every operator-facing function lives in exactly one of these eight slots. No fluid mix. If a thing doesn't fit, it's either misplaced or it needs a smaller surface (a column, a row-action, a tab).
# Slot Use when Don't use when
1Keel (top strip)Truly global: ⌘K search, 5 launch chips, health pill, user.Anything view-specific.
2Side nav (route)A dedicated, frequently-visited operator surface.Anything reachable from existing routes via a row or a drawer.
3Pre-selection stripAffordances to start work on this view (Vendor ▾, "+ Receive Row").Anything that requires a row to already exist.
4Selection strip (primary + More tray)Status-aware primary action + 2-4 secondary verbs in a tray for the selected row(s).Reference content or supporting context.
5Grid column / inline cellData scanned and edited as part of a row.Anything that's not a per-row attribute.
6Context Drawer tabReferential or supporting context for the active entity (history, balances, suggestions, tools, validation, allocations, photos, manifest, etc.).Anything that needs to be edited from inside the drawer (drawer is read-only; clicks route out).
7Cmd-K palette (+ alias)Power command, rare admin action, anything that benefits from being typed by name.Daily high-frequency work.
8HotkeyTruly repeated motion (commit, duplicate, filter, drawer, focus).Anything that isn't repeated.
If a new function fits two slots, prefer the smaller surface (column > tab > palette). If it fits none, it's almost certainly redundant.

Tabs are read-only inside the drawer. Clicks route to that surface and select the row. ⌘← restores prior state. Drawer is closed by default on queue views; opens on entity selection.
The five questions to ask before adding any new control
  1. Is this a per-row attribute the operator scans and edits? → Add it as a grid column. Done.
  2. Is this the obvious primary action for the row at its current status? → Selection strip primary. Done.
  3. Is this a less-frequent but row-specific action? → Selection strip More ▾ tray. Done.
  4. Is this reference data or supporting context about the active entity? → Drawer tab. Done.
  5. Is this a rare admin / power command? → Cmd-K palette with alias. Done.
If a function answers no to all five, it's almost certainly not needed. If it answers yes to multiple, use the smaller surface. "It needs its own panel on the page" is what we're trying to never say again.
