Vendor Payouts · default spec: Due reason is a first-class visible column — "consigned lot depleted (B-201)" / "net 30 from PO-003 receipt 5/02" / "terms · 2 partials so far" — so the operator never has to wonder why a bill is due. Open column (Amount − Paid) makes partial payouts legible at a glance. Next-action column previews the status-aware primary, same pattern as Orders. Old VendorBillTools panel (manual bill, void payout) is gone from canvas — those tools move into the drawer's Tools tab on a vendor.

Vendor Payouts · selection spec: Status-aware primary cascade: open → Approve, approved → Schedule, scheduled → Pay, partial → Pay remaining, paid → primary disabled (tray: Reverse / Void). Due reason tab tells the operator why a consignment bill is due in plain language — pulled from queries.relationshipDrawer or vendorBills.dueReason. Trace ribbon at the bottom mirrors the PO Linked-intake trace pattern: PO → receipt → batch → bill → payouts. Consignment tab shows the remaining-units-vs-sold math for sellout-driven payables. Tools tab (manual bill creation / void payout) is reachable via the drawer's More ▾ tab list (hidden in 7-tab default to keep ribbon clean).
What batch 3 establishes
  • Status-aware primary cascades across the order lifecycle (draft→Confirm→Post→Allocate→Mark Pay/F-up) and the payable lifecycle (open→Approve→Schedule→Pay).
  • "Next action" column on queue grids — the same verb the primary will become when that row is selected. Removes the "what do I do with this" question before the operator even clicks.
  • Quick Ledger as the money front door — append rows, preview impact, commit per row. Allocation tools / discount / refund / reverse move into drawer tabs on selected payments.
  • Negative money-in auto-flips to buyer credit with inline label and impact preview — already in QuickLedgerGrid; no behavior change.
  • Due reason as a first-class column on Vendor Payouts, plus a dedicated Due reason drawer tab explaining sellout / net-terms / down-payment / scheduled-event triggers in plain language.
  • Trace ribbons on payable Due reason tab and PO Linked-intake tab share the same pattern: PO → receipt → batch → bill → payouts, clickable end to end.
  • No always-on side panels on Payments or Vendor Payouts. Allocation tools, manual-bill creation, payout voiding, FIFO Allocate all live inside the drawer's tabs.

Batch 4 next: Fulfillment, Connectors, Recovery, Closeout, and the new Reports route — all in sketch fidelity per scope. Then the consolidated spec doc gets written, reviewed, and handed to writing-plans.
