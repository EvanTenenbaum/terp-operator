Why aging matters

Aging inventory drives margin erosion (price drops to clear) and consigned payable urgency. Use this report alongside Vendor Performance to spot which vendors keep delivering slow-moving categories.

Reports · selection spec: Selecting a row in the report grid promotes that slice to the selection strip and exposes a status-aware primary that's specific to the report (Aging inventory → Open lots in Inventory; Vendor performance → Open vendor card; Revenue → Open client card). What batch 4 establishes (closes the wireframe scope)
  • Status-aware primary cascades now defined for Fulfillment (Pack → Print labels → Mark fulfilled), Connectors (Route), Recovery (Retry / Reverse / disabled), Closeout (Fix unsafe → Lock → Archive).
  • Inline cells beat strips wherever rows need editing — pack qty/weight/bag on Fulfillment, routeTo/operatorNotes on Connectors, etc.
  • Recovery is a queue + drawer: command grid by default, 9 specialty tabs (Reversal · Snapshot · Source map · Find/Replace · Correction · Backup · Markers · System · History) in the drawer. The 3-band cockpit is gone.
  • Closeout treats period as an entity: current period in a top card, archived in the grid, selecting either activates the identity ribbon and opens the drawer.
  • Reports is a chip-row picker + grid + drawer Definition tab: seven cross-entity reports without a new lane per report. Selection within a report routes to the source lane via a status-aware primary.
