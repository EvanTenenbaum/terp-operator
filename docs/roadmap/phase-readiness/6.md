# Phase 6 Readiness — Decide and Reports

Date: 2026-05-12
Status: ready after core loop surfaces are stable

## Purpose

Give the owner fast daily truth and calm report projections without turning the dashboard into a chart wall or a second workflow engine.

## Required Reading

- `docs/product/work-loops.md` Loop 7
- `docs/product/capability-registry.md` CAP-021, CAP-025, CAP-015, CAP-016, BE-006, BE-007
- `docs/design/replication-playbook.md` R7, R9, R10, R11, R12

## Prerequisites

- Reports route shell exists from Phase 0b.
- Source rows for Sales, Payments, Payables, Inventory, Closeout, and Connectors are stable.
- Report math expectations are written before implementation.

## Scope

| Item | Product reason | Capability IDs | Recipe |
| --- | --- | --- | --- |
| Dashboard Today Focus | Owner sees the few decisions that matter now. | CAP-021 | R7, R12 |
| Reports route | Dedicated projection lane for repeated review. | CAP-021 | R6, R7 |
| Source-row drilldowns | KPI/report values prove themselves with rows. | CAP-021, CAP-025 | R12 |
| Report exports | Deterministic CSV/packet outputs. | CAP-021 | R10 |
| Suggestion/search freshness decision | Add projection freshness only if direct queries are insufficient. | CAP-015, CAP-016 | R9 |

## Backend Dependencies

Packet D is optional unless direct client-side aggregation becomes slow or stale. Do not introduce generated projection tables before evidence demands them.

## Out Of Scope

- Decorative realtime charting.
- AI-written financial advice.
- Mobile dashboard.
- Excel export unless explicit operator need appears.

## Feature Flag

Reports route may remain visible after Phase 0b; individual reports can ship behind per-report flags until math fixtures pass.

## Pre-Flight Checks

```bash
pnpm typecheck

pnpm audit:product-roadmap
pnpm build
```

## Acceptance Evidence

- Dashboard metrics have plain-language definitions.
- Each KPI/report value drills to source rows or a documented no-row formula.
- Report math matches seeded fixture totals.
- Reports use calm table-first layout with optional small charts only where they clarify.
- Export headers are deterministic.

## Smoke Test

1. Open Dashboard.
2. Drill into cash/files, payables, receivables, aging inventory, and pending work.
3. Open Reports.
4. Run at least one revenue, payables, inventory aging, and category report.
5. Export CSV and compare deterministic headers.

## Rollback

Disable individual reports. Dashboard baseline remains available.
