Date: 2026-05-12
Status: final gate phase

## Purpose

Turn the integrated product into a releasable operator console: keyboard-stable, accessible, fast enough, vocabulary-clean, drift-checked, and deployable on owner-controlled infrastructure.

- Phases 0a-6 complete or explicitly deferred in registry with owner-visible reason.

## Files Likely Touched

- hotkey/focus infrastructure

## Backend Dependencies

All backend commands must be represented in command catalog, role gates, audit journal, reversal matrix, and frontend parity. No backend mutation ships as "hidden for later" without registry disposition.

## Out Of Scope

- New product scope.
- New large UI redesign.
- New backend domain not already in registry.
- Nice-to-have theme work that slows release.

## Feature Flag

Decide which flags become default-on. Remove dead fallback code only after soak passes and rollback plan is documented.

## Acceptance Evidence

- All core journeys can be completed keyboard-only or with documented exception.
- Drawer/focus state persists per route/entity without corrupting active selection.
- Customer-facing outputs pass internal-field deny-list checks.
- Command parity audit is green.
- Product roadmap audit is green.
- Seeded performance targets are measured or explicit blockers are recorded.
- Deployment docs are current for DigitalOcean same-origin app.
