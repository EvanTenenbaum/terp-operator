# Phase 4 Readiness — Sell Follow-Through

Date: 2026-05-12
Status: ready after Phase 1 and Phase 3 foundations

## Purpose

Carry a confirmed sale through posting, fulfillment, connector review, and client ledger support without making operators hunt across unrelated pages.

## Acceptance Evidence

- Posted order can be followed to fulfillment without losing selected order context.
- Customer-safe order output hides internal fields.
- Fulfillment line can record actual qty/weight and bag assignment inline.
- Connector request shows source, requested items, review history, route, and safety banner.
- No connector code path writes ledger consequences directly.

## Smoke Test

1. Open posted order.
2. Allocate to fulfillment.
3. Record weigh/pack on a line.
4. Assign bag and print label status.
5. Route a connector request to Sales or Fulfillment.
6. Verify connector history shows route/review and no direct ledger mutation.

## Rollback

Disable view subflags. Keep connector backend bridge disabled by default until dedicated tests pass.
