# Phase 0b Readiness — Canvas Grammar Behind Flag

Date: 2026-05-12
Status: ready after Phase 0a

## Purpose

Introduce the canonical TERP Agro canvas grammar behind a reversible flag: Keel, SideNav groups, Identity Ribbon, Context Drawer, focus states, and a Reports route shell. This is the frame that keeps future frontend work coherent.

## Required Reading

- `docs/design/spec.md` sections 1-3, 10, 19, 22
- `docs/design/replication-playbook.md`
- `docs/product/north-stars.md`
- `docs/product/capability-registry.md`
- Phase 0a readiness evidence

## Prerequisites

- Phase 0a complete.
- Existing routes still pass smoke.
- No unresolved conflict in `docs/roadmap/open-disagreements.md`.

- New report math.
- New backend projection tables.
- New drawer tabs beyond minimal default scaffolding.
- Rebuilding every view in one PR.

## Feature Flag

Required. Use one reversible UI flag for canvas grammar until Phase 7. Default can be off during implementation and on for QA previews.

## Pre-Flight Checks

```bash
pnpm typecheck
pnpm audit:parity
pnpm audit:product-roadmap
pnpm build
```

## Acceptance Evidence

- Flag off: current app behavior remains available.
- Flag on: Keel, SideNav groups, Identity Ribbon, and Context Drawer render on all main routes.
- Drawer supports `closed`, `peek`, `standard`, `wide`, and `focus` states or a documented equivalent matching the spec.
- Operators can minimize/expand context without losing grid focus.
- Reports route exists as a projection lane but does not invent new business logic.

## Smoke Test

1. Toggle canvas flag on.
2. Open each route.
3. Select a row where seeded data exists.
4. Open/close drawer with keyboard and click controls.
5. Switch focus mode and confirm grid space expands.
6. Toggle flag off and confirm old route shell still works.

## Rollback

Disable the feature flag. If needed, revert canvas components without touching backend or data.

## Estimated PR Size

Medium to large. Keep primitives and route rewiring separate if review risk grows.
