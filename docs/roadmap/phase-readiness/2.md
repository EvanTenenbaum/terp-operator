# Phase 2 Readiness — Procurement, Receiving, and Global Finder

Date: 2026-05-12
Status: ready after Phase 1 or in parallel with backend Packet A

## Purpose

Make New PO, Receive Inventory, and Product Finder fast, clear, and separate. Buying before product arrives must not be confused with receiving physical product.

## Required Reading

- `docs/product/work-loops.md` Loops 1 and 2
- `docs/product/capability-registry.md` CAP-002, CAP-003, CAP-005, CAP-011, CAP-019, CAP-023, CAP-028, BE-002, BE-003, BE-009
- `docs/design/replication-playbook.md` R1, R2, R4, R9, R10, R12

## Prerequisites

- Context Drawer primitive exists.
- Purchase Orders and Intake commands are parity-green.
- Product Finder baseline behavior is captured by tests before extraction.

## Acceptance Evidence

- New PO starts in one obvious place and first line entry is grid-native.
- Receive Inventory remains separate from PO creation.
- Vendor receipt totals match selected intake row subtotals.
- Finder behavior remains at least as capable as before extraction.
- Inventory location/status/ownership UI only exposes commands that exist or clearly marks backend gap.

## Smoke Test

1. Create draft PO.
2. Add two PO lines.
3. Approve PO.
4. Receive one line into draft intake.
5. Preview receipt from selected intake rows.
6. Use Finder from Sales and Inventory frames; compare result counts to baseline fixture.

## Rollback

Disable Finder extraction flag or revert PO/Intake drawer changes. Draft data remains governed by existing commands.
