# Phase 1 Readiness — Customer-Centered Selling

Date: 2026-05-12
Status: ready after Phase 0b

## Purpose

Make New Sale incredibly easy to start while keeping the spreadsheet-first order-building model. Sales should begin from customer context, inventory slice, or remembered item text without forcing a form/wizard.

## Required Reading

- `docs/product/work-loops.md` Loop 3

- `docs/design/spec.md` Sales sections and status-aware action table

- Phase 0b canvas flag works.
- Sales route has Identity Ribbon and Context Drawer primitives.
- Existing sales commands are parity-green.

## Acceptance Evidence

- New Sale starts from Sales with one obvious action and first editable line focused.
- Customer context is visible without scrolling away from the grid.
- Finder filters/slices inventory by availability, category, vendor, tags, price band, and aging where current data supports it.
- Customer-safe output hides cost, margin, vendor floor, internal notes, and pricing rule details.
- Confirm/Post remains status-aware and row-native.

## Smoke Test

1. Start a new sale.
2. Select customer.
3. Slice inventory using at least three filters.
4. Add a line from Finder.
5. Confirm or price the order.
6. Generate customer-safe output.
7. Verify internal fields are absent.
